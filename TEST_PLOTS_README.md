# Test Plots Directory

This directory contains diagnostic performance plots generated by the integration tests in `core/tests/integration_tests.rs`.

## Purpose

These plots are **purely diagnostic** and intended for developer review during testing and development. They are **not publication-ready figures** but provide a quick visual check of filter performance against GPS ground truth.

## Contents

When you run the integration tests, the following diagnostic plots are generated:

- `dead_reckoning.png` - Pure INS dead reckoning performance (baseline)
- `ukf_closed_loop.png` - UKF with full-rate GNSS updates
- `ukf_degraded_gnss.png` - UKF with reduced GNSS update rate (5s intervals)
- `ekf_closed_loop.png` - EKF with full-rate GNSS updates
- `ekf_degraded_gnss.png` - EKF with reduced GNSS update rate (5s intervals)
- `eskf_closed_loop.png` - Error-State KF with full-rate GNSS updates

## Plot Details

Each plot shows:
- **2D Haversine Horizontal Error** (red line) - Distance between estimated and GPS position
- **Altitude Error** (blue line) - Vertical position error
- **GPS Horizontal Accuracy** (green dots) - Reported GPS accuracy (when available)
- **GPS Vertical Accuracy** (magenta dots) - Reported GPS vertical accuracy (when available)

## Usage

1. **Running tests**: The plots are automatically generated when you run integration tests:
   ```bash
   cargo test --package strapdown-core --test integration_tests
   ```

2. **Reviewing plots**: Open the PNG files in this directory to visually inspect filter performance

3. **Comparing filters**: Use the plots to compare different filter types (UKF, EKF, ESKF) or degradation scenarios

## Cleanup

**This directory and its contents can be safely deleted at any time.** The plots will be regenerated the next time you run the integration tests.

To remove:
```bash
rm -rf test_plots/
```

## Notes

- Plots are created using the `plotters` crate in Rust
- The plotting functionality is conditionally compiled based on the "plotting" feature (enabled by default)
- For publication-ready figures or more detailed analysis, use the Python scripts in the repository instead
